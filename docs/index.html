performance-snapshot-auto-runner

Performance Snapshot Auto Runner

Description

Checks for Event Notifications in Unifier that are generated when Performance Snapshot Business Processes are created, then runs a User Defined Report, wbs_perf, for the project that the snapshot BP was created, then updates the snapshot BP with line items generated from the report, then finally sends the snapshot BP to the terminal step.

Built using Typescript 4.6 as an ESM package

Installation

Requires NodeJS v16+

It is Recommended to install PM2 5.2.0+ to run this application as it has been built with this in mind. PM2 Docs

ESBuild is required to bundle the application

Usage

npm scripts:

  • npm run start - runs index.ts via ts-node
  • npm run test - runs Jest test suites
  • npm run build - runs esbuild to package the application to standard javascript.

Build Process

The application leverages ESBuild to transpile from typescript and bundle all dependencies into one file.

##Testing Tests have been written with Jest

Unit tests can be found adjacent to source files with the convention name.test.ts

Integration tests can be found in the tests/ folder in the source code.

a .env file must be created and loaded by Jest for the tests to complete successfully. The Required variables are as follows: UNIFIER_USER_TOKEN=Unifier Integration user auth token BASE_URL=base url of unifier environment (without /unifier) to be used for testing purposes TEST_PROJ_ID=Project ID to use in tests TEST_RP=Reporting Period to be used to create Performance Snapshots TEST_ACCEPTED_BP=The ID of a BP that has been manually accepted must be created in the test project in the test environment.
